Troubleshooting App (Flutter)

A cross-platform Flutter app built to support a UK post-mix beverage company’s field technicians. The app streamlines troubleshooting workflows, organizes service data, and provides offline-first access to repair procedures.

Overview

Designed to enhance field support operations, this app helps technicians quickly find solutions to common issues in post-mix beverage systems. Built using Flutter and Dart, it integrates a RESTful API backend for centralized management of technical guides, parts information, and client data.

What I Did

I developed the app from scratch using Flutter, focusing on a clean and intuitive UI for field engineers. Features include interactive troubleshooting flows, spare-parts lookup, starred favorites, offline caching, and push notifications for updates.

The project also includes an admin dashboard (web-based) that allows internal staff to upload new troubleshooting content, manage part details, and monitor app usage analytics.

Key Features

  • Guided troubleshooting flows with step-by-step instructions.
  • Offline access for technicians working in low-connectivity areas.
  • Dynamic folders for parts, towers, valves, and components.
  • Favorites system for saving frequently used guides.
  • Firebase push notifications for updates and new content.

Tech Stack

Flutter · Dart · REST API · Firebase · FCM · Local Storage · Provider (State Management)
