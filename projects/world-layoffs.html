World Layoffs Data Cleaning

A data-cleaning project focused on transforming a raw dataset of global company layoffs into a structured, analysis-ready format. I used SQL to identify and remove duplicates, fix inconsistent values, and handle nulls to ensure a high-quality, reliable dataset.

What I Did

I designed and executed a systematic data cleaning workflow using MySQL. The process began with creating a dedicated staging table to protect the original dataset, followed by applying cleaning operations such as removing duplicate entries, standardizing industry and country names, converting data types, and handling null values responsibly.

My main objective was to make the dataset consistent, accurate, and easy to use for further exploratory data analysis (EDA) or visualization. The final cleaned table was optimized for reporting and ready for integration into analytical pipelines.

Key Contributions

  • Developed SQL scripts to remove duplicate and inconsistent records.
  • Standardized categorical fields and enforced proper data types.
  • Reviewed and cleaned null and missing values for analytical accuracy.
  • Produced a fully cleaned table (layoffs_staging2) for downstream analysis.
  • Ensured all transformations preserved data integrity and traceability.
